# Feature Specs

One file per feature. Describe the intent and acceptance criteria before implementation starts.

## Naming convention
`short-feature-name.md`

## Template
```
## Feature: <title>
**Status:** planned | in-progress | shipped

### Why
(user problem or goal)

### What
(behaviour from the user's perspective)

### Acceptance Criteria
- [ ] criterion one
- [ ] criterion two

### Out of Scope
- thing we are NOT doing
```
